# Configuration Reference

All configuration is through environment variables in the `.env` file. Run `bin/setup` to generate one with secure defaults.

## Required

| Variable | Description | Example |
|----------|-------------|---------|
| `INBOXED_DOMAIN` | Domain for inbound SMTP | `mail.example.com` |
| `INBOXED_BASE_URL` | Public URL of the API | `https://mail.example.com` |
| `DASHBOARD_URL` | Public URL of the dashboard (used for CORS and OAuth redirects) | `https://mail.example.com` |
| `SECRET_KEY_BASE` | Rails secret key (128 hex chars) | *(auto-generated by bin/setup)* |
| `POSTGRES_PASSWORD` | PostgreSQL password | *(auto-generated by bin/setup)* |

## Setup & Auth

| Variable | Default | Description |
|----------|---------|-------------|
| `INBOXED_SETUP_TOKEN` | *(none)* | One-time token for creating the admin account via `/setup` |
| `REGISTRATION_MODE` | `closed` | `open` (anyone), `invite_only` (token required), `closed` (admin only) |
| `TRIAL_DURATION_DAYS` | `7` | Trial days for new orgs when registration is open (0 = permanent) |
| `GITHUB_CLIENT_ID` | *(empty)* | GitHub OAuth app client ID (enables GitHub login) |
| `GITHUB_CLIENT_SECRET` | *(empty)* | GitHub OAuth app client secret |

## Outbound SMTP

System emails for verification, password reset, and invitations. Without this, users are auto-verified and password reset is unavailable.

| Variable | Default | Description |
|----------|---------|-------------|
| `OUTBOUND_SMTP_HOST` | *(empty)* | SMTP relay host (e.g., `smtp.resend.com`) |
| `OUTBOUND_FROM_EMAIL` | *(empty)* | From address for system emails |

## Ports

| Variable | Default | Description |
|----------|---------|-------------|
| `INBOXED_WEB_PORT` | `80` | Dashboard HTTP port |
| `INBOXED_API_PORT` | `3000` | API HTTP port |
| `INBOXED_SMTP_PORT` | `587` | SMTP submission port |
| `INBOXED_SMTPS_PORT` | `465` | SMTPS (implicit TLS) port |
| `INBOXED_MCP_PORT` | `3001` | MCP server port |

## Email & Storage

| Variable | Default | Description |
|----------|---------|-------------|
| `EMAIL_TTL_HOURS` | `168` (7 days) | Email retention period |
| `INBOXED_MAX_EMAILS_PER_PROJECT` | `10000` | Maximum emails stored per project |
| `INBOXED_MAX_MESSAGE_SIZE_MB` | `25` | Maximum email size in megabytes |

## MCP Server

| Variable | Default | Description |
|----------|---------|-------------|
| `INBOXED_MCP_KEY` | *(empty)* | API key for MCP server to access the API |

## Feature Flags

| Variable | Default | Description |
|----------|---------|-------------|
| `INBOXED_FEATURE_HOOKS` | `true` | Enable webhook catching endpoints |
| `INBOXED_FEATURE_FORMS` | `true` | Enable form catching endpoints |
| `INBOXED_FEATURE_HEARTBEATS` | `true` | Enable heartbeat monitoring endpoints |

## Rate Limits

| Variable | Default | Description |
|----------|---------|-------------|
| `RATE_LIMIT_API` | `300` | API requests per minute per API key |
| `RATE_LIMIT_ADMIN` | `60` | Admin requests per minute per IP |
| `RATE_LIMIT_HOOK` | `120` | Hook catch requests per minute per token |
| `RATE_LIMIT_HOOK_GLOBAL` | `1000` | Hook catch requests per minute globally |
| `RATE_LIMIT_AUTH` | `5` | Failed auth attempts per 5 minutes per IP |

## SMTP Server (Inbound)

| Variable | Default | Description |
|----------|---------|-------------|
| `SMTP_HOST` | `0.0.0.0` | SMTP bind address |
| `SMTP_PORTS` | `2525` | Internal SMTP port (mapped via docker-compose) |
| `SMTP_MAX_CONNECTIONS` | `4` | Maximum concurrent SMTP connections |
| `SMTP_TLS_CERT` | *(empty)* | Path to TLS certificate (enables STARTTLS) |
| `SMTP_TLS_KEY` | *(empty)* | Path to TLS private key |

## Database & Infrastructure

| Variable | Default | Description |
|----------|---------|-------------|
| `DATABASE_URL` | *(set in docker-compose)* | Full PostgreSQL connection URL |
| `REDIS_URL` | *(set in docker-compose)* | Redis URL for ActionCable and caching |
| `RAILS_MAX_THREADS` | `5` | Puma thread count |
| `RAILS_LOG_LEVEL` | `info` | Log verbosity: `debug`, `info`, `warn`, `error`, `fatal` |
| `JOB_CONCURRENCY` | `5` | Solid Queue worker concurrency |

In production, API request logs are structured JSON via lograge:

```json
{"method":"GET","path":"/api/v1/inboxes","status":200,"duration":12.34,"request_id":"abc-123","ip":"10.0.0.1"}
```
